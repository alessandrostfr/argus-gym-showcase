# Roadmap

## Estado actual

Argus Gym tiene una base amplia de producto:

- backend FastAPI;
- frontend Next.js;
- auth;
- calendario;
- entrenamiento live;
- Studio;
- comunidad;
- chat;
- coach mode;
- admin;
- analytics;
- media.

El siguiente objetivo no es añadir más features sin control, sino preparar el proyecto para portfolio, capturas limpias y una demo pública revisada. Actualmente existe una demo inicial en https://argusgym.vercel.app/.

---

## Próximos pasos

### 1. Demo pública segura

- datos ficticios;
- usuarios demo;
- entorno separado;
- capturas limpias;
- enlaces desde portfolio.

### 2. Despliegue en VPS

- Docker;
- reverse proxy;
- HTTPS;
- PostgreSQL;
- backups;
- monitorización.

### 3. Pulido visual

- revisar pantallas principales;
- reducir fricción;
- reforzar mobile-first;
- mejorar empty states;
- mejorar onboarding.

### 4. Case study en portfolio

- problema;
- solución;
- arquitectura;
- stack;
- capturas;
- decisiones técnicas;
- demo.

### 5. Posible app móvil futura

Flutter queda como fase posterior, cuando la web esté estable y defendible.

---

## No prioritario ahora

- añadir features grandes sin cerrar demo;
- reabrir módulos ya funcionales sin motivo;
- publicar código fuente;
- complicar infraestructura antes de tener demo estable.
